# 트리(Tree)

### 개념

- **계층적인 자료를 표현하는 데 이용되는 자료구조**
  - 예) 조직도, 컴퓨터의 디렉터리 구조 등
  - `리스트`, `스택`, `큐` 등의 `선형 자료구조`는 자료들이 직선과 같이 나열되어 있는 구조
  - 자료의 계층적인 관계를 표현하고 싶은 경우 선형 구조는 적합X 
- **트리는 한 개 이상의 노드로 이루어진 유한 집합**
  - 이들 중 하나의 노드는 `루트(root) 노드`이며 나머지 노드들은 `서브트리(subtree)`
  - **서브트리는 다시 루트와 서브트리로 구성**
  - 루트와 서브트리는 `간선(edge)`으로 연결
  - 노드들 간에는 부모 관계, 형제 관계, 조상과 자손 관계가 존재하는데 이는 인간의 관계와 동일
  - `차수(degree)`: 어떤 노드가 가지고 있는 자식 노드의 개수
    - 트리의 차수는 트리가 가지고 있는 노드의 차수 중에서 가장 큰 차수
  - `레벨(level)`: 트리에서 각 층의 번호
    - 트리의 높이는 트리가 가지고 있는 최대 레벨



### 이진트리(Binary tree)

```
공집합이거나 루트와 왼쪽 서브트리, 오른쪽 서브트리로 구성된 노드들의 유한집합으로 정의
```

- **모든 노드가 2개의 서브트리를 가지고 있는 트리**
  - 서브트리는 `공집합`일 수도 있기 때문에 이진 트리의 노드는 최대 2개의 자식 노드가 존재할 수 있고 모든 노드의 차수는 2 이하가 됨
  - 이진트리는 순서가 존재하며 왼쪽 서브트리와 오른쪽 서브트리로 구별
  - **이진트리의 서브트리도 이진트리의 성질을 만족** `(순환적 정의)` 
- 이진트리의 성질
  - 노드의 개수가 n이면 간선의 개수는 n-1
  - 높이가 h인 이진트리의 경우 최소 h개의 노드를 가지며 최대 2^h - 1개의 노드를 가짐
  - n개의 노드를 가지는 이진트리의 높이는 최대 n, 최소 log2(n + 1)

- 이진트리의 분류
  - `포화 이진트리(full binary tree)`: 트리의 각 레벨에 노드가 꽉 차 있는 이진트리
  - `완전 이진트리(complete binary tree)`: 높이가 h일 때 레벨 1부터 h - 1까지는 노드가 모두 채워져 있고 마지막 레벨 h에서는 왼쪽부터 오른쪽으로 노드가 순서대로 채워져있는 이진트리



### 순회

- **포인터를 이용하여 구현한 트리는 루트 노드를 가리키는 포인터만 있으면 트리 안의 모든 노드들에 접근 가능**
  - 이는 연결된 구조인 `연결 리스트(Linked List)`와 유사
- 트리가 가지고 있는 자료를 순차적으로 순회하는 것은 이진트리에서 중요한 연산
  - **이진트리의 순회(traversal)는 모든 노드를 한 번씩 방문하여 노드가 가지고 있는 데이터를 목적에 맞게 처리하는 것**
  - 트리는 순환적으로 정의되기 때문에 트리 순회 알고리즘은 `순환 기법`을 사용
    - **이진트리에서 전체 트리와 서브트리의 구조는 동일하기 때문**
- 이진트리의 순회방법은 루트와 왼쪽 서브트리, 오른쪽 서브트리 중 **루트를 언제 방문하느냐**에 따라 구분
  - `전위순회(preorder traversal)`: **V**LR
  - `중위순회(inorder traversal)`: L**V**R
  - `후위순회(postorder traversal)`: LR**V**



### 구현

- **배열을 이용하는 방법**
  - 이진트리의 높이가 k이면 최대 2^k - 1개의 공간을 연속적으로 할당한 다음 이진트리의 번호대로 노드를 저장
  - `인덱스`만 알면 노드의 부모나 자식 관계 파악이 쉬움
  - 완전 이진트리가 아닌 일반적인 이진트리인 경우에는 기억 공간의 낭비가 심해짐 예) 경사 이진트리
